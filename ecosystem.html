Ecosystem architecture

A full economy structure, explained clearly.

Veyloria is designed as a layered economy ecosystem, not just a collection of individual commands. The goal is to make every major part of the bot feel connected: setup creates the foundation, profiles give members identity, banking stores value, income creates activity, markets move items, businesses create long-term growth, and support keeps the experience understandable.

This page is written as the public product overview for the ecosystem. It explains what each system is supposed to do, how it fits into the wider economy and why it matters for a Discord community that wants a more polished economy experience. The tone is intentionally practical and clear, so server owners can understand the product without needing to read the full command directory first.

Product logic

The ecosystem is split into four practical layers.

Each layer has a different purpose. This keeps the website, the support server and the command directory easier to understand. A new user can start with basic identity and money commands, while a server owner can focus on setup, logs, permissions and economy control.

Foundation layerConfiguration, prefix, language, logs, module toggles, permissions, server setup and staff-only safety tools.
User economy layerProfiles, wallet, bank, income, jobs, rewards, inventory, achievements, reputation and visible progression.
Expansion layerMarket, trading, auctions, businesses, properties, resources, crafting, clans, racing, farms and pets.
Product layerPremium access, official links, support tickets, documentation, policies, status information and community guidance.

System modules

Twelve major areas for the Veyloria experience.

The modules below are written as ecosystem descriptions, not as final implementation promises. They show how the command catalogue can be presented on the website while keeping the language mature, detailed and product-focused.

Module01

Core Configuration

The core layer is the administrative foundation of Veyloria. It covers prefix control, language direction, currency configuration, economy channels, logs, module toggles, command availability and server-level safety settings. This section exists so server owners can shape Veyloria around their community before members begin using economy features.

How it should feel in the product

Server owners and trusted administrators use this layer to configure how the bot behaves, where economy activity is allowed and how sensitive activity is logged. It should feel practical rather than technical, giving staff a clear first-run path and reducing setup mistakes.

Example command direction

v!setupv!prefixv!setcurrencyv!modulev!setlogv!setmarketv!cooldowns
Module02

Profile and Identity

The identity layer gives each member a readable economic presence. Profiles, titles, badges, reputation, history, rank, achievements and comparison tools turn raw money into a long-term identity that other members can understand at a glance.

How it should feel in the product

This is the layer that makes progression visible. It should show who a user is inside the economy, what they have built, how trusted they are and how far they have progressed without making the interface feel crowded.

Example command direction

v!startv!profilev!statsv!titlev!badgev!reputationv!rankv!history
Module03

Wallet and Banking

The banking layer manages the most important financial actions: wallet balance, bank storage, deposits, withdrawals, transfers, statements, net worth, limits and interest-style progression. It should be simple enough for new members and structured enough for advanced systems to depend on it.

How it should feel in the product

A stable banking layer protects the rest of the economy from feeling random. The user should understand where their money is, what is liquid, what is stored, what can be transferred and how their net worth is calculated.

Example command direction

v!balancev!walletv!bankv!depositv!withdrawv!payv!statementv!networth
Module04

Income and Careers

The income layer gives users controlled ways to earn currency through daily rewards, weekly rewards, work, jobs, shifts, careers, tasks and cooldown-based progression. It should reward activity without turning the economy into uncontrolled inflation.

How it should feel in the product

Career systems can grow from simple commands into a structured progression path where better jobs, experience and time investment produce stronger rewards. Cooldowns and server configuration help keep this balanced.

Example command direction

v!dailyv!weeklyv!workv!jobv!jobsv!careerv!paydayv!tasks
Module05

Market and Commerce

The market layer moves Veyloria beyond basic earning. Shops, buying, selling, market listings, auctions, bids, trades, receipts and price visibility give users a reason to interact with each other instead of only farming commands.

How it should feel in the product

A strong market system should feel transparent and readable. Members need to know what they are buying, who they are trading with, what the price means and how to review activity after the transaction.

Example command direction

v!shopv!buyv!sellv!marketv!marketbuyv!auctionv!bidv!trade
Module06

Business and Estates

The business layer is where long-term economy growth becomes more serious. Business creation, upgrades, collection, property ownership, rent, estates and asset growth can turn the economy into a slower strategic system rather than a quick reward loop.

How it should feel in the product

This section should be presented as advanced progression. Users who already understand wallet, bank and market systems can begin building assets that create identity, income and status over time.

Example command direction

v!businessv!businesscreatev!businesscollectv!businessupgradev!propertyv!propertiesv!propertybuyv!rent
Module07

Resources and Crafting

The resource layer gives the economy physical texture. Mining, fishing, farming, harvesting, crafting, recipes, energy and inventory actions help transform the bot from a bank simulator into an economy with materials, production and item movement.

How it should feel in the product

Crafting is especially useful because it creates demand for items and gives market systems more purpose. Resources can feed businesses, quests, upgrades, events and special collections.

Example command direction

v!inventoryv!itemv!minev!fishv!farmv!harvestv!craftv!recipes
Module08

Clans, Factions and Competition

The social competition layer gives groups a reason to organize. Factions, clans, clan creation, wars, raids, reputation, leaderboards and team-based rivalry can make the economy feel alive without relying only on individual balances.

How it should feel in the product

This layer should be controlled carefully because competitive systems can create strong engagement but also need clear rules, cooldowns and anti-abuse protection. It works best when tied to leaderboards, rewards and visible progression.

Example command direction

v!factionv!factioncreatev!clanv!clancreatev!raidv!warv!topv!compare
Module09

Garage, Racing and Transport

The garage layer adds lifestyle progression through vehicles, racing, upgrades, routes and transport-based status. It gives users another long-term collection path that can connect to market value, events and public competition.

How it should feel in the product

Racing and transport systems should feel optional but meaningful. They can support leaderboards, betting limits, car collections, route bonuses and future prestige paths without becoming mandatory for every user.

Example command direction

v!garagev!carbuyv!racev!racejoinv!routev!vehiclev!upgradev!leaderboard
Module10

Farming, Pets and Genetics

The living ecosystem layer includes farming, pets, pet shops, eggs, feeding, training, genetics and collection-based progression. It adds a softer but still strategic side to the economy, useful for players who enjoy growth systems and collectibles.

How it should feel in the product

Pets and farms can become strong retention systems because they give members something to maintain over time. They can also connect to crafting, events, competitions, market items and profile identity.

Example command direction

v!farmv!harvestv!petv!petshopv!petbuyv!petfeedv!pettrainv!incubate
Module11

Audit, Admin and Economy Control

The control layer is for staff and owners. It covers economy logs, item creation, price edits, event starts, reward roles, tax settings, bank limits, backups, blacklists and global safety actions needed to keep a large economy under control.

How it should feel in the product

These tools should be protected behind staff permissions. They are not part of the public game loop; they exist so the server can recover from mistakes, adjust balance and review suspicious activity.

Example command direction

v!auditv!itemcreatev!pricesetv!eventstartv!rewardrolev!taxsetv!backupv!blacklist
Module12

Premium, Support and Public Trust

The product layer surrounds the bot with a proper support experience. Premium information, perks, support tickets, documentation, community guidelines, status notices and official links make the bot feel safer and more reliable for communities.

How it should feel in the product

This is what makes Veyloria feel like a product rather than a command pack. Users should know where to get help, how premium works, what rules apply and which links are official.

Example command direction

v!premiumv!vipperksv!ticketv!reportv!suggestv!statusv!docsv!invite
For members

Start with profile, wallet and bank.

Members should understand their account before using advanced modules. A clear profile, stable balance view and simple banking flow make every later system easier to learn.

For server owners

Configure before launching publicly.

Owners should prepare economy channels, logs, permissions, roles, support paths and command availability before members begin using the economy heavily.

For staff

Keep support and audit visible.

Staff should have clear places for ticket review, bug reports, suspicious economy activity, moderation logs and configuration notes so issues can be handled without confusion.
