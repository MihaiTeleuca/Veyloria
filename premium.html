Payment automation

Yes, automatic payments are possible, but not from static HTML alone.

GitHub Pages can host the premium page and payment buttons, but it cannot safely receive webhooks, store secrets or grant Discord roles by itself. For automatic activation, Veyloria needs a payment processor and a small trusted automation layer.

1

Checkout

The user clicks a premium button and pays through Stripe Payment Links, Stripe Checkout or Discord's native premium app purchase flow.

2

Webhook

The payment platform sends a payment or subscription event to a backend endpoint when the checkout succeeds, renews, fails or is cancelled.

3

Discord access

The backend verifies the event and grants or removes the premium role through Discord, or stores premium access for the bot to check.

Recommended automation flow: GitHub Pages premium.html → Stripe Payment Link / Discord Premium App checkout → Webhook endpoint on Cloudflare Workers, Vercel, Render or similar → Verify payment event → Match buyer to Discord user ID → Grant Premium role or enable premium access → Remove access if subscription is cancelled or payment fails

Options

Choose the payment model before going live.

There are three realistic ways to handle premium. The right choice depends on whether you want a simple launch, full automation or native Discord checkout.

OptionWhat it doesAutomation levelBest for
Manual checkoutPayment button sends users to Stripe, PayPal, Ko-fi or support. Staff grants the role manually after checking proof.LowFastest first launch with the least technical setup.
Stripe webhookStripe handles checkout, then a webhook endpoint grants or removes premium access automatically.HighProfessional external payments, subscriptions and automatic role updates.
Discord Premium AppUsers purchase through Discord's native app monetization flow, and the app checks entitlement access.HighMost native Discord experience, if the app is eligible and configured for monetization.
ImportantDo not put secret keys, bot tokens, Stripe secret keys or webhook secrets inside GitHub Pages files. Anything in a static website is public to users.

What the premium page can do today

It can show plans, benefits, rules, support expectations and payment buttons. It can safely link to external checkout pages and explain how premium is activated.

What needs backend automation

Automatic role assignment, subscription cancellation handling, failed payment removal, Discord user matching and audit logs need a backend or native Discord monetization integration.

Best simple launch

Use Stripe Payment Links first, then manually grant premium roles through support while the backend automation is being built and tested.

Best professional launch

Use Stripe subscriptions or Discord Premium App monetization with verified webhooks and automatic premium access handling.
